# EQUIPMENT (装備) YAML スキーマ定義

武器・防具・アクセサリー・ツール等の「装備してステータスを向上させるアイテム」を表現します。
また、装備アイテムでも「使用することで特殊な効果（コマンド実行など）を発動する」用途を想定します。

## スキーマ定義

| キー                            | 型                   | 必須 | デフォルト     | 説明                                           |
|:------------------------------|:--------------------|:--:|:----------|:---------------------------------------------|
| `schemaVersion`               | Integer             | ○  | -         | スキーマのバージョン（2026-01-19時点は `1`）                |
| `id`                          | String              | ○  | -         | テンプレートID（例: `iron_sword`）※大文字小文字の区別あり        |
| `category`                    | String              | ○  | -         | カテゴリ（`EQUIPMENT`）                            |
| `name`                        | String              | ○  | -         | ゲーム内に表示される名前（§ または & の色コード利用可能）              |
| `icon`                        | String              | ○  | -         | Bukkit Material名（例: `IRON_SWORD`）            |
| `rarity`                      | String              | ○  | -         | rarity（アイテム共通定義参照）                           |
| `customModelData`             | Integer             | ×  | Null      | クライアント側リソースパック用のモデルデータID（任意）                 |
| `lore`                        | List<String>        | ×  | emptyList | 説明文（§ または & の色コード利用可能）                       |
| `baseStats`                   | Object              | ○  | -         | 装備の基本ステータス（後述）                               |
| `baseStats.attack`            | Number              | ×  | 0         | 攻撃力（単位はゲーム内定義に従う）                            |
| `baseStats.defense`           | Number              | ×  | 0         | 防御力                                          |
| `baseStats.speed`             | Number              | ×  | 0         | 速度（例: 移動速度補正など。解釈は実装側に委ねる）                   |
| `baseStats.critChance`        | Number              | ×  | 0         | クリティカル率（例: `0.05` = 5%）                      |
| `baseStats.critDamage`        | Number              | ×  | 0         | クリティカルダメージ（例: `0.25` = +25%）                 |
| `scaling`                     | Object              | ×  | Null      | レベル等による成長設定（後述）                              |
| `scaling.levelCap`            | Integer             | ×  | -         | 成長が適用される上限レベル（例: 20）                         |
| `scaling.perLevel`            | Object              | ×  | -         | 1レベルあたりの増加量（後述）                              |
| `scaling.perLevel.attack`     | Number              | ×  | 0         | 1レベルあたり攻撃力増加                                 |
| `scaling.perLevel.defense`    | Number              | ×  | 0         | 1レベルあたり防御力増加                                 |
| `scaling.perLevel.speed`      | Number              | ×  | 0         | 1レベルあたり速度増加                                  |
| `scaling.perLevel.critChance` | Number              | ×  | 0         | 1レベルあたりクリ率増加（例: `0.002` = +0.2%）             |
| `scaling.perLevel.critDamage` | Number              | ×  | 0         | 1レベルあたりクリダメ増加                                |
| `requirements`                | Object              | ×  | Null      | 装備条件（後述）                                     |
| `requirements.level`          | Integer             | ×  | 0         | 必要レベル（0 の場合は条件なし）                            |
| `requirements.classes[]`      | List<String>        | ×  | emptyList | 装備可能クラスのリスト（空の場合は制限なし）                       |
| `durability`                  | Object              | ×  | Null      | 耐久（後述）                                       |
| `durability.max`              | Integer             | ○  | -         | 最大耐久値（`1` 以上）                                |
| `durability.loss`             | Object              | ×  | -         | 消耗量（後述）                                      |
| `durability.loss.hit`         | Integer             | ×  | 0         | 攻撃（ヒット）1回あたりの消耗量                             |
| `durability.loss.block`       | Integer             | ×  | 0         | 防御（ブロック）1回あたりの消耗量                            |
| `sockets`                     | Object              | ×  | Null      | ソケット設定（後述）                                   |
| `sockets.count`               | Integer             | ○  | -         | ソケット数（`0` 以上）                                |
| `sockets.types[]`             | List<String>        | ×  | emptyList | 受け入れ可能なソケット種別（空の場合は制限なし）                     |
| `allowedEnchants[]`           | List<String>        | ×  | emptyList | 許可するエンチャントのリスト（空の場合は制限なし）                    |
| `flags`                       | Object              | ×  | Null      | 運用フラグ（後述）                                    |
| `flags.untradeable`           | Boolean             | ×  | false     | trueでトレード不可                                  |
| `flags.unstackable`           | Boolean             | ×  | false     | trueでスタック不可（例: 1個ずつ管理する装備）                   |
| `flags.soulbindOnEquip`       | Boolean             | ×  | false     | trueで装備時にソウルバインド（帰属）する（実装側で解釈）               |
| `onUse`                       | Object              | ×  | Null      | 使用時効果（後述）                                    |
| `onUse.commands[]`            | List<String>        | ×  | emptyList | 使用時に実行するコマンド（複数可）。プレースホルダ等のルールは実装側に委ねる       |
| `onUse.cooldownTicks`         | Long                | ×  | 0         | 使用クールダウン（tick）。Minecraftの慣習として 20 tick = 1 秒 |
| `nbtTags`                     | Map<String, String> | ×  | emptyMap  | NBTタグ（任意）。キー/値の解釈は実装側に委ねる                    |

### requirements.classes[]
装備可能クラスを指定します（列挙は実装側のクラス定義に従う）。

例:
- `WARRIOR`
- `PALADIN`

### sockets.types[]
ソケットの種別を指定します（列挙は運用で拡張される想定）。

例:
- `OFFENSE`
- `DEFENSE`
- `UTILITY`

### allowedEnchants[]
許可するエンチャントを指定します（Bukkit/Spigotの `Enchantment` に対応するキー想定）。

例:
- `SHARPNESS`
- `UNBREAKING`

### 参照（ref）
Equipment はアイテムテンプレートの一種です。
他DBから Equipment を参照する場合は、アイテム共通の参照規約に従い `item:` prefix を使用します。

- 例: `ref: item:iron_sword`

※ Equipment 固有キー内で他DBを参照する場合（例: Buff付与などを拡張する場合）は、参照先DBの prefix/aliases を使用してください。
（例: Buffは `buff:`（aliases: `bf`）、LootTableは `loot_table:`（aliases: `lt`））

## YAML 例

### 例1: 武器（成長・装備条件・耐久・ソケット・使用効果）

```yaml
schemaVersion: 1
id: iron_sword
category: EQUIPMENT
name: "アイアンソード"
material: "IRON_SWORD"
customModelData: 12001
rarity: UNCOMMON
lore:
  - "§7鍛冶師が作った堅牢な剣"
  - "§aATK +{stats.attack}"

baseStats:
  attack: 8
  defense: 0
  speed: 0
  critChance: 0.05
  critDamage: 0.25

scaling:
  levelCap: 20
  perLevel:
    attack: 0.4

requirements:
  level: 3
  classes: ["WARRIOR", "PALADIN"]

durability:
  max: 250
  loss:
    hit: 1
    block: 1

sockets:
  count: 1
  types: ["OFFENSE"]

allowedEnchants: ["SHARPNESS", "UNBREAKING"]

flags:
  untradeable: false
  unstackable: true
  soulbindOnEquip: true

onUse:
  commands:
    - "say {player} swings the sword!"
  cooldownTicks: 10

nbtTags:
  myplugin: "IRON_SWORD"
  bind: "SOULBOUND"
```

### 例2: アクセサリー（耐久なし・成長なし・制限なしの最小寄り）

```yaml
schemaVersion: 1
id: lucky_charm
category: EQUIPMENT
name: "&aラッキーチャーム"
material: "AMETHYST_SHARD"
rarity: RARE
lore:
  - "&7身に付けると幸運が訪れると言われている。"

baseStats:
  attack: 0
  defense: 2
  speed: 0
  critChance: 0.02
  critDamage: 0.10

flags:
  untradeable: false
  unstackable: true
```
